⚙️ WordPress Settings API – Explained
📘 What Is the Settings API?

The Settings API is a framework in WordPress that allows developers to create settings pages, sections, and fields in a secure and standardized way. It handles data validation, saving, and retrieval automatically.

🔑 Core Functions
  • register_setting($group, $option_name) – Registers a setting with a group.
  • add_settings_section($id, $title, $callback, $page) – Adds a section to a settings page.
  • add_settings_field($id, $title, $callback, $page, $section) – Adds a field to a section.
  • settings_fields($group) – Outputs hidden fields for security and group association.
  • do_settings_sections($page) – Outputs all sections and fields for a page.
🧪 How It Works in Your Plugin
  1. Register the setting:
    register_setting('tct_settings_group', 'tct_options');
    This tells WordPress to save the tct_options array when submitted via the tct_settings_group form.
  2. Output the form:
    <form method="post" action="options.php">
    <?php settings_fields('tct_settings_group'); ?>
    This sets up the form with the correct security fields and submission target.
  3. Save and retrieve:
    WordPress automatically saves the data to the database.
    Use get_option('tct_options') to retrieve it in your theme or plugin.
💡 Why Use the Settings API?
  • ✅ Built-in security (nonces, sanitization)
  • ✅ Consistent admin UI
  • ✅ Automatic saving and retrieval
  • ✅ Scalable for complex settings pages
🛠️ Optional: Full Section & Field Example
// Register setting
register_setting('my_group', 'my_option');

// Add section
add_settings_section('my_section', 'My Section', null, 'my_page');

// Add field
add_settings_field('my_field', 'My Field', 'render_my_field', 'my_page', 'my_section');

// Render field
function render_my_field() {
    $value = get_option('my_option');
    echo '<input type="text" name="my_option" value="' . esc_attr($value) . '" />';
}

This modular approach is ideal for large settings pages with many fields.
